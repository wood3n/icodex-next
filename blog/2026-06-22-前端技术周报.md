本期前端技术周报统计 2026-06-15 至 2026-06-22 的官方更新，重点覆盖 pnpm 安全修复、npm 12 破坏性预览，以及构建、解析和组件文档工作流的变化。

<!--truncate-->

## 包管理与供应链

本分类参考来源：[pnpm releases](https://github.com/pnpm/pnpm/releases)、[npm CLI releases](https://github.com/npm/cli/releases)、[Yarn releases](https://github.com/yarnpkg/berry/releases)。

### pnpm

> 参考来源：[pnpm 11.8.0](https://github.com/pnpm/pnpm/releases/tag/v11.8.0)、[pnpm 10.34.4](https://github.com/pnpm/pnpm/releases/tag/v10.34.4)

pnpm 于 6 月 18 日发布 11.8.0 和 10.34.4，修复恶意 `configDependencies`、lockfile alias 与 `patch-remove` 造成路径穿越或越界写入的问题。

使用仓库提交的 `pnpm-lock.yaml`、`pnpm-workspace.yaml` 或 `nodeLinker: hoisted` 时应优先升级。11.8.0 还新增 `pnpm install --dry-run`、Node.js package map、按 workspace 拆分 SBOM；其中 `pnpm run --no-bail` 现在只要任一脚本失败就返回非零状态，依赖旧退出码的 CI 需要调整。

### npm

> 参考来源：[npm CLI 12.0.0-pre.1](https://github.com/npm/cli/releases/tag/v12.0.0-pre.1)

npm 于 6 月 19 日发布 12.0.0-pre.1，预览版默认拒绝 git 与远程 tarball 依赖、要求显式批准安装脚本，并收紧 Node.js 版本和配置解析。

该版本仅适合迁移验证：`allow-git`、`allow-remote` 默认改为 `none`，未知 `.npmrc` 配置、缩写参数和单横线长参数会直接报错，根项目 `preinstall` 也提前到依赖安装前执行。内部脚手架、私有 git 依赖和安装脚本策略应在 npm 12 稳定版前完成兼容检查。

### Yarn

> 参考来源：[Yarn 4.17.0](https://github.com/yarnpkg/berry/releases/tag/%40yarnpkg/cli/4.17.0)

Yarn 于 6 月 15 日发布 4.17.0，新增 package map 生成能力，并允许按 npm scope 配置 `npmMinimalAgeGate`。

package map 面向 Node.js 新的包解析工作流，当前应先在隔离环境验证生成结果与 PnP、`node_modules` linker 的差异。按 scope 设置最小发布时间门槛后，团队可以对公共 registry 保持供应链缓冲，同时为受控私有 scope 配置不同策略。

## 构建与模块解析

本分类参考来源：[Rspack releases](https://github.com/web-infra-dev/rspack/releases)、[Rolldown releases](https://github.com/rolldown/rolldown/releases)。

### Rspack

> 参考来源：[Rspack 2.1.0-beta.0](https://github.com/web-infra-dev/rspack/releases/tag/v2.1.0-beta.0)

Rspack 于 6 月 18 日发布 2.1.0-beta.0，内置 SWC loader 可直接启用 React Compiler，生产构建默认开启无副作用函数分析。

默认 tree shaking 行为变化可能改变产物内容，应对使用 `#__NO_SIDE_EFFECTS__`、`pureFunctions`、barrel exports 和跨模块纯函数的代码做快照与运行时回归。React Compiler 配置仍属于 beta 验证范围，建议只在测试分支开启 `jsc.transform.reactCompiler`。

### Rolldown

> 参考来源：[Rolldown 1.1.2](https://github.com/rolldown/rolldown/releases/tag/v1.1.2)

Rolldown 于 6 月 18 日发布稳定版 1.1.2，调整 `resolve.tsconfigPaths` 的项目归属判定，并修复 worker threads 崩溃与跨 chunk 初始化问题。

显式在 `tsconfig.include` 中列出的 `.vue`、`.svelte` 文件现在可正确使用 `paths`；不属于任何 TS project 的文件则不再继承最外层 `tsconfig` 的 alias。Vite 8 使用同一解析器，monorepo、project references 和 solution-style 配置升级后应检查别名解析是否与 tsserver 一致。

## 组件文档与测试

本分类参考来源：[Storybook releases](https://github.com/storybookjs/storybook/releases)。

### Storybook

> 参考来源：[Storybook 10.4.5](https://github.com/storybookjs/storybook/releases/tag/v10.4.5)、[Storybook 10.4.6](https://github.com/storybookjs/storybook/releases/tag/v10.4.6)

Storybook 于 6 月 15–16 日发布稳定版 10.4.5 和 10.4.6，修复 CSF3/CSF4 混用时重复注入 annotations，并允许 story 与 meta 部分覆盖 globals。

混用不同 CSF 版本或依赖全局 toolbar、locale、theme 参数的组件库可直接跟进稳定版。升级后应回归 globals 合并顺序、preview annotations、交互测试和静态构建；本次同时升级了 esbuild，构建产物或插件兼容异常也应纳入检查。
