## AI 编程工具

### Codex

OpenAI 在 5 月 21 日的 [ChatGPT Release Notes](https://help.openai.com/en/articles/6825453-chatgpt-release-notes) 中集中发布了一组 Codex 更新，关键词是：上下文输入更丰富、长任务目标模式正式可用、浏览器反馈更细、远程锁屏运行更完整。

主要变化包括：

1. Appshots：Codex macOS App 可以通过快捷键把一个应用窗口附加到当前线程，包含截图和可用文本。对调试 UI、理解设计稿、分析本地应用状态很有用。
2. Goal mode 正式可用：可以把目标和验收标准交给 Codex，让它持续推进长任务。覆盖 Codex App、IDE extension 和 CLI。
3. In-app browser annotations：浏览器标注更精细，适合前端页面、样式、交互状态的反馈。
4. Locked computer use：符合条件的 Mac Computer Use 用户可以让 Codex 在 Mac 锁屏后继续远程、安全地运行，但仍受 Computer Use 区域限制约束。
5. Browser use improvements：包含高级标注模式、更快的资源提取、只读 JavaScript 上下文、标签分组体验和可靠性改进。

这次更新对使用 Codex 做真实开发任务的人影响比较大。过去给 AI 编程助手补上下文经常要靠长 prompt、截图描述或手动复制终端输出，现在 Codex 更接近“能看到你正在看的东西，并围绕目标持续推进”的工作流。

### GitHub Copilot

GitHub Copilot 上周有多条 Changelog 更新，重点都围绕 Web Chat、VS Code 里的模型选择，以及 IDE 生态透明度。

5 月 20 日，Copilot Web 更新了 [可用模型列表](https://github.blog/changelog/2026-05-20-updates-to-available-models-in-copilot-on-web)。GitHub 移除了 Web Chat 里的所有 Gemini 模型，以及 GPT-5.2 Codex、GPT-5.4 nano 等部分模型，只保留跨价位的 OpenAI 和 Claude 模型。官方给出的理由是减少模型列表复杂度，保证更稳定的一致质量。

同一天，Copilot Chat on web 上线了 [semantic issue search](https://github.blog/changelog/2026-05-20-semantic-issue-search-in-copilot-chat/)。它可以用自然语言在仓库 issue 中做语义查找，不再只依赖精确关键词和手动过滤。对维护大型开源仓库或内部平台仓库的人来说，这更适合做 issue triage、重复问题发现、需求归类。

VS Code 侧，Copilot 的 [auto model selection](https://github.blog/changelog/2026-05-20-auto-model-selection-now-routes-based-on-your-task-in-vs-code/) 也做了更新：它会结合模型健康度、利用率和任务类型，在推理、代码生成、问题诊断、工具编排等维度上选择更合适的模型。用户可以悬停查看实际使用了哪个模型。

5 月 21 日，GitHub 还宣布 [Copilot for Eclipse 开源](https://github.blog/changelog/2026-05-21-github-copilot-for-eclipse-is-open-source)。这对 Eclipse 生态和企业 Java 团队比较有意义：AI IDE 插件不再只是封闭分发，社区可以直接查看实现、提交贡献和排查问题。

## 模型与助手平台

### Gemini

Google 在 5 月 19 日的 I/O 2026 期间发布了 [Gemini App 新一轮更新](https://blog.google/innovation-and-ai/products/gemini-app/next-evolution-gemini-app/)，同时在 [开发者主题公告](https://developers.googleblog.com/en/all-the-news-from-the-google-io-2026-developer-keynote/) 中总结了 Gemini 3.5 系列、Antigravity 和 AI Studio 相关能力。

面向普通用户，Gemini App 的方向是更 agentic：新的 UI、主动 daily briefs，以及名为 Gemini Spark 的全天候 agent。Google 还提到 macOS 桌面应用会集成 Gemini Spark，使其能够在本地机器上操作，并增强语音能力。

面向开发者，Google 宣布 Gemini 3.5 系列模型，并升级 Antigravity 这个 agent-first 开发平台。结合 Google AI Studio、Gemini API、Android Studio 和企业平台来看，Google 正在把 Gemini 从“聊天助手”推进到“模型 + agent 平台 + 开发工具”的组合。

这类更新对开发者的直接影响是：Gemini 不再只是一个模型选择项，而是越来越多地出现在 IDE、API、搜索、Android、企业 agent 平台里。后续选型时不能只比较模型分数，还要看工具调用、上下文、部署入口和产品集成深度。

### ChatGPT

ChatGPT Release Notes 同样在 5 月 21 日记录了 Codex 相关更新，所以本周 ChatGPT 对开发者最重要的变化其实是 Codex 能力进入更完整的跨端工作流。

如果只看 ChatGPT 主产品，上周没有看到新的通用模型发布；但 5 月 14 日发布的 [Codex remote access from the ChatGPT mobile app](https://help.openai.com/en/articles/6825453-chatgpt-release-notes) 已经开始把 Codex 接入 ChatGPT 移动端：用户可以从手机继续线程、回答问题、审批动作、查看 Codex 的发现，并在已连接的 Mac host 间切换。

对开发者来说，ChatGPT 移动端不再只是聊天入口，而逐渐成为远程监督本地 agent 的入口。这个方向和 Codex 的 Goal mode、locked computer use 放在一起看，说明 OpenAI 在把“AI 编程助手”从一次性问答推进到持续工作会话。

## 企业与合规

### Claude

Anthropic 在 5 月 21 日的 [Claude Release Notes](https://support.claude.com/en/articles/12138966-release-notes) 中发布了 Claude Compliance API integrations。官方描述是 Claude 现在可以接入更多安全与合规工具，让 IT 和安全团队像治理其他企业应用一样治理 Claude 平台和产品套件。

这次更新不是模型能力升级，而是企业落地能力补齐。对企业团队来说，AI 产品能不能用，很多时候不只取决于模型是否聪明，还取决于审计、合规、权限、数据治理能不能接入已有安全体系。

如果团队正在推进 Claude、Claude Code 或 Claude Enterprise 的内部使用，这类合规集成会影响采购和上线速度。普通个人用户感知不强，但企业管理员、安全团队和平台工程团队会更关注。

## 本周观察

上周 AI 产品的主线很清楚：AI 助手正在从“模型能力竞争”进入“工作流接管能力竞争”。

Codex 在补上下文、目标、远程运行和浏览器协作；Copilot 在收敛模型选择，同时增强 issue 语义检索和 IDE 生态；Gemini 把 agent、桌面、API、AI Studio、Antigravity 串在一起；Claude 则继续补企业治理和合规集成。

对开发者来说，后续选 AI 工具不能只问“哪个模型更强”，还要看三个问题：

1. 它能不能稳定拿到项目上下文。
2. 它能不能在 IDE、浏览器、终端、移动端之间持续工作。
3. 它能不能被团队治理，包括权限、审计、费用和合规。
