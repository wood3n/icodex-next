## AI 模型与平台

### GPT-5.6 与 GPT-Live

> 参考来源：[GPT-5.6: Frontier intelligence that scales with your ambition](https://openai.com/index/gpt-5-6/)

OpenAI 于 7 月 9 日发布 GPT-5.6 系列，包含 Sol（旗舰）、Terra（均衡）与 Luna（经济）三款模型。

| 模型 | 定位 | ALE 分数 | ExploitBench2 | OSWorld 2.0 | 输入价格 | 输出价格 |
|------|------|----------|---------------|-------------|----------|----------|
| Sol | 旗舰 | 53.6 | 73.5% | 62.6% | $5/M | $30/M |
| Terra | 均衡 | — | — | — | — | — |
| Luna | 经济 | — | — | — | — | — |

Sol 在 Agents' Last Exam 中以 53.6 分领先 Claude Fable 5 达 13.1 分，在 OSWorld 2.0 以 62.6% 超越 Opus 4.8 的同时减少 85% 输出 token。

新增 Programmatic Tool Calling，模型可编写轻量程序编排多工具，无需每步回传大模型。ultra 模式默认协调四个并行 agent。API 定价 Sol $5/$30（输入/输出每百万 token），支持 prompt caching 与缓存读取 90% 折扣。同日发布的 GPT-Live 为全双工语音模型，在 ChatGPT 与 API 中可用。

### Cursor 3.11

> 参考来源：[Cursor changelog](https://cursor.com/en-US/changelog)

Cursor 于 7 月 10 日发布 3.11 版本，新增 Side Chats 功能，可在不中断主 agent 对话的前提下创建并行的侧聊会话，每条侧聊为完整持久会话并可回引至主线程。Conversation Search 通过 Cmd+K 在 agent 对话记录中搜索，本地索引可扩展至数千条对话。

Cloud Agent Hooks 新增 beforeSubmitPrompt、afterAgentResponse、afterAgentThought 等钩子，用于观察和控制 agent 对话流程。项目与仓库选择器重新设计，支持在选取器内直接创建项目并连接 GitHub/GitLab/Azure DevOps。

### Codex

> 参考来源：[Codex changelog](https://learn.chatgpt.com/docs/changelog)

Codex 于 7 月 9 日并入 ChatGPT 桌面版应用（macOS/Windows）。现有用户保留项目与设置，可设为默认视图。新增侧边栏内审查 GitHub PR、跨仓库协同、内联 Markdown 与代码编辑等能力，Computer Use 性能因 GPT-5.6 而增强。

Codex CLI 0.144.0 新增写入应用审批模式（声明只读操作、写入操作需确认）和 MCP 工具交互式认证。0.143.0 将远程插件默认启用，支持系统代理及 PAC/WPAD，新增 Amazon Bedrock GPT-5.6 模型接入。

## 编译语言

### TypeScript 7.0

> 参考来源：[Announcing TypeScript 7.0](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0/)

TypeScript 于 7 月 8 日发布 7.0 正式版，核心编译器以 Go 重写，全量构建提速 8-12 倍。VS Code 项目构建从 125.7 秒降至 10.6 秒，Sentry 从 139.8 秒降至 15.7 秒，内存占用减少 6%-26%。新增实验性 `--checkers` 与 `--builders` 参数控制并行度，`--watch` 模式移植自 Parcel 文件监听器。

6.0 中的弃用项在 7.0 中变为硬错误：移除 `target: es5`、`downlevelIteration`、`moduleResolution: node`/`classic` 等选项。模板字面量类型保留 Unicode 码点。7.0 不提供程序化 API（预计 7.1 加入），使用 Angular/Vue/Svelte/Astro 模板的项目暂无法利用 7.0，建议继续使用 6.0。兼容包 `@typescript/typescript6` 提供 `tsc6` 实现并排运行。

## 运行时

### Bun

> 参考来源：[Rewriting Bun in Rust](https://bun.com/blog/bun-in-rust)

Bun 于 7 月 8 日宣布从 Zig 迁移至 Rust，535,496 行 Zig 代码转为约 100 个 Rust crate。迁移由 64 个并发 Claude 实例协同完成，历时 11 天、6,778 次提交。

| 指标 | Zig (v1.3.14) | Rust (v1.4.0) | 变化 |
|------|---------------|---------------|------|
| 代码量 | 535,496 行 | ~100 crate | — |
| 二进制大小 | 88 MB | 70 MB | ↓ 20% |
| Bun.serve 吞吐 | 基准 | — | ↑ 4.8% |
| 2000 次构建后内存 | 6,745 MB | 609 MB | ↓ 91% |
| unsafe 代码占比 | — | ~4% | 主要为 C/C++ 指针操作 |

v1.3.14 为最后 Zig 版本，v1.4.0（Rust 版）可通过 `bun upgrade --canary` 获取。Bun 于 2025 年 12 月被 Anthropic 收购。

### Node.js 26.5.0

> 参考来源：[Node v26.5.0](https://nodejs.org/en/blog/release/v26.5.0)

Node.js 于 7 月 8 日发布 26.5.0 Current 版本。新增 `Blob.prototype.textStream()` 以 ReadableStream 流式返回 Blob 文本内容，实验性 `--experimental-import-text` 标志支持 ESM 中导入 `.txt` 为字符串。perf_hooks 新增按事件循环迭代采样的延迟属性。

`ReadableStreamTee` 从 Web Streams 标准中暴露，支持流克隆。TLS 层新增协商密钥交换组上报（含后量子 ML-KEM）。maglev 编译器启用 riscv64 架构。依赖更新：undici 8.7.0、nghttp3 1.17.0、sqlite 3.53.3。

## 包管理

### npm 12.0.0

> 参考来源：[npm v12.0.0](https://github.com/npm/cli/releases/tag/v12.0.0)

npm 于 7 月 8 日发布 12.0.0 稳定版。预发布阶段的 linked install strategy 毕业为正式特性。破坏性变更包括移除 `npm shrinkwrap`、`npm adduser`、`star`/`stars`/`unstar` 命令，`allow-git` 和 `allow-remote` 默认改为 `none`，未知配置项直接抛出错误而非告警。

`npm view --json` 始终返回数组，默认 license 从 ISC 改为空字符串，Twitter/Freenode 资料字段移除。Node 引擎要求 `^22.22.2 || ^24.15.0 || >=26.0.0`。workspaces 支持从预发布转为稳定。

## 构建工具

### Rolldown 1.1.5

> 参考来源：[Rolldown 1.1.5](https://github.com/rolldown/rolldown/releases/tag/v1.1.5)

Rolldown 于 7 月 8 日发布 1.1.5 稳定版。新增执行顺序敏感的 import 绑定读取检测、`sourcemapFileNames` 配置项、`--configLoader=native` CLI 选项以及插件 API 描述性元数据。

14 项修复涵盖：可观察后的重复资源文件名稳定化、循环声明符 tree-shaking、HMR 回退全量重载、增量构建关系更新、`sideEffects: false` 的副作用门控、Vite resolve 中 `browser: false` 映射返回空对象等。多项 MagicString 边界情况修复。

## UI 组件

### Shadcn/UI

> 参考来源：[Shadcn/UI changelog](https://ui.shadcn.com/docs/changelog)

Shadcn/UI 于 7 月发布 typeset 样式系统，通过单个 CSS 文件和三个 CSS 变量（`--typeset-size`、`--typeset-leading`、`--typeset-flow`）控制 HTML 与 Markdown 文本排版，原生支持流式渲染，无需额外包或配置层。

Base UI 正式成为新项目默认组件库，当前 1.6.0 版本、周下载量 600 万以上。`npx shadcn init` 默认使用 Base UI，Radix 通过 `-b radix` 可选保留，两套体系均可独立受支持。

## 格式化与 Lint

### Oxlint 1.73.0

> 参考来源：[Oxlint v1.73.0](https://raw.githubusercontent.com/oxc-project/oxc/refs/heads/main/crates/oxc_linter/CHANGELOG.md)

Oxlint 于 7 月 6 日发布 1.73.0。新增 `explicit-timer-delay`、`no-confusing-array-with`、`no-unreachable-loop` 三条规则，`unicorn/filename-case` 新增 `lowercase` 与 `screamingSnakeCase` 选项。类型感知 lint 支持按规则计时。

约 12 项性能优化包括：早期退出快路径、规则集惰性计算、O(1) 查表层类型集合以及向精确节点类型收缩规则分发。

### ESLint 10.7.0

> 参考来源：[ESLint v10.7.0 released](https://eslint.org/blog/2026/07/eslint-v10.7.0-released/)

ESLint 于 7 月 10 日发布 10.7.0 次要版本。`max-nested-callbacks` 新增 `checkConstructorCallCallbacks` 选项，将传入构造器的回调计入嵌套深度。`preserve-caught-error` 新增 `errorClassNames` 选项，支持自定义错误类。`no-compare-neg-zero` 现在提供替换建议。`radix` 规则支持计算属性访问的 Number.parseInt 检测。

13 项修复覆盖 `eqeqeq` 静态模板字面量处理、`no-invalid-regexp` 假阳性、`prefer-numeric-literals` 边界情况等。CI 新增 Node.js 26 支持。
